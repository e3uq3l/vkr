3. Исходные данные для создания программной системы

3.1. Перечень решаемых задач
\renewcommand\labelenumi{\theenumi)}
\begin{enumerate}
	\item проанализировать предметную деятельность учебно-методического центра;
	\item разработать концептуальную модель и базу данных;
	\item спроектировать архитектуру (веб-интерфейс + десктоп-приложение);
	\item реализовать и протестировать программную систему.
\end{enumerate}


3.2. Входные и выходные данные
\begin{enumerate}
	\item {Входные данные:}
сведения о слушателях, справочники (статусы, программы, кафедры, источники финансирования), параметры фильтрации, Excel-файлы для импорта.
	\item {Выходные данные:}
таблица слушателей с фильтрацией, формы редактирования, статистические отчёты, журнал событий, управление пользователями.
\end{enumerate}



4. Содержание работы (по разделам)

 4.1. Введение.
 
 4.2. Анализ предметной области.
 
 4.3. Техническое задание: основание для разработки, назначение разработки,
 требования к программной системе, требования к оформлению документации.
 
 4.4. Технический проект: общие сведения о программной системе, проект
 данных программной системы, проектирование архитектуры программной системы, проектирование пользовательского интерфейса программной системы.
 
 4.5. Рабочий проект: спецификация компонентов и классов программной системы, тестирование программной системы, сборка компонентов программной системы.
 
 4.6. Заключение.
 
 4.7. Список использованных источников.
